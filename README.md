# Hytale-MMO-Modded

Modded Hytale MMO server. 48 mods covering RPG leveling, classes, pets, mounts, dungeons, quests, masks, cosmetics, and more.

### Dungeon Instance Levels
| Instance | Level Range |
|----------|-------------|
| YUNG's Skeleton Dungeon | 50 to 65 |
| Major Dungeons D01 | 55 to 70 |
| Major Dungeons D02 | 70 to 85 |
| Major Dungeons D03 | 85 to 100 |

Dungeon instance levels are static. `InstanceLevelConfig.json` supports `LevelMin`, `LevelMax`, `EntityOverrides`, `DisableRPGLeveling`, and `ExactIdMatch` per instance. There is no per instance XP rate (`ZoneRateExp` only exists on zone entries). The global `RateExp` applies everywhere. The HUD color codes based on how the player's level compares to the instance range.

YUNG's HyDungeons is a content pack (no Java code) that uses Hytale's native instance system. RPG Leveling recognizes it via `ExactIdMatch: true`. Major Dungeons is a Java plugin with boss system, NPC shops (Dark Coins), and loot. Uses instance IDs `MJ_Instance_D01/D02/D03` with `ExactIdMatch: false` since the game appends UUIDs. Boss HP and damage are set by Major Dungeons, not scaled by RPG Leveling.

## Permissions (LuckPerms)

Setup guide: `configs/LuckPerms/README_SETUP.md`

LuckPerms hooks into Hytale's `PermissionsModule`, so permissions set with `/lp` commands work with all mods including AdminUI and SimpleClaims.

### Default Group (All Players)
```
/lp group default permission set simpleclaimsparty.* true
/lp group default permission set simpleclaimschunk.* true
/lp group default permission set mounts.wolf_black true
```

### SimpleClaims Fix
Regular players couldn't accept group invites because the accept command requires the `simpleclaimsparty.invite-accept` permission. The wildcard grant above fixes this.

### Zone Mount Permissions
```
/lp creategroup zone2
/lp group zone2 permission set mounts.bear_polar true
/lp creategroup zone3
/lp group zone3 permission set mounts.rex_cave true
/lp creategroup zone4
/lp group zone4 permission set mounts.dragon_frost true
```

## Loot4Everyone

Configured in game through the `/lootconfig` UI. No external config file.

### Recommended Settings
| Setting | Value | Notes |
|---------|-------|-------|
| IsLootRandom | `true` | Randomized drops per player |
| CanPlayerBreakLootChests | `false` | Prevents griefing |
| IsMessageAppear | `true` | Notifies when loot is available |
| IsParticlesAppear | `true` | Shows sparkles on loot chests |
| LootResetMode | Timer | So dungeons can be rerun |
| NextLootResetInterval | ~2 hours | Stops farming but allows replays |

## Blook's Bandits and Pirates

Both mods come with their own NPCs, drops, and spawn rules. No extra configuration needed.

### Bandits (Zone 1)
- Spawn in Forests and Swamps at night (19:00 to 21:00)
- Assassin (55%), Brute (5%), Rabbit decoy (40%)
- Drop Fabric Scraps (100%) and Bandit Mask (22%)
- Uses beacon spawning

### Pirates (Zone 2 and 3)
- Human pirates on Zone 2 shores during the day (7:00 to 18:00), includes Sailors, Gunners, and a Captain
- Skeleton pirates on Zone 3 shores, spawn all day
- Drop Boom Powder (100%), Fabric (100%), Cutlass (10%)
- Uses world spawning

### Saurian's Variety Dinosaurs
Content mod with 22+ dinosaur skin variants, custom sound effects, and a new Triceratops. Spawns naturally through its own role definitions. Nothing to configure.
